Capabilities

What the Companion Does

Built specifically for the LT&I workflow — from the initial support email through diagnosis to resolution and team learning.

🔍

Diagnostic Conversations

Structured EXPLORE → DIAGNOSE → RESOLVE phases. The Companion asks follow-up questions instead of jumping to conclusions, because users often report symptoms, not root causes.

📚

Knowledge Base Search

Semantic search across Moodle documentation, TRU FAQs, WordPress content, and internal guides. Relevant documentation is automatically pulled into every diagnostic conversation.

🔗

Moodle URL Parsing

Paste any moodle.tru.ca URL and the Companion automatically extracts the course ID, module type, and activity — giving instant context for the conversation.

📦

Course Context Upload

Upload .mbz course backups, screenshots, or saved Moodle HTML pages. The Companion analyzes course structure, settings, and configuration without needing admin access.

🩺

Course Health Checks

Automated detection of common configuration issues: completion tracking gaps, gradebook inconsistencies, generic activity names, format mismatches, and more.

📋

Case Tracking

Save diagnostic sessions as cases with tags, difficulty ratings, and full conversation history. Build institutional memory your team can search and learn from.

📊

Analytics Dashboard

See common issues at a glance: trending categories, difficulty distribution, resolution times, top Moodle modules, and a 12-week timeline of team activity.

✉️

Draft Reply Generator

Generate professional replies ready to paste into TeamDynamix. Choose the audience (instructor, student, or admin) and tone (professional, friendly, or brief).

🤝

Sharing & Export

Share a conversation with a colleague via a direct link. Export sessions as Markdown for training materials. Export all cases as CSV for reporting.

Methodology

How It Works

The Companion follows a structured diagnostic methodology — it resists jumping to solutions until it understands the problem.

🔍
EXPLORE
Gather info, ask questions
→
🧠
DIAGNOSE
Analyze, rank causes
→
✅
RESOLVE
Fix steps, draft reply

For each support issue, the Companion:

  1. Searches the knowledge base for relevant Moodle documentation and past cases
  2. Parses any Moodle URLs to identify the course, module, and activity involved
  3. Suggests checks for the technologist to perform and questions to ask the end user
  4. Cross-references the knowledge base and similar past cases before diagnosing
  5. Ranks possible causes by likelihood and provides step-by-step resolution
  6. Offers to draft a reply for the user, save the case for team learning, or create a KB article

User Guide

Using the Companion

The tool has three main tabs: Diagnose, Search, and Cases.

💡 Diagnose Tab — Diagnostic Conversations

This is where you investigate support issues. Start by describing the problem as you understand it.

1

Describe the issue

Type the support issue into the message box. Include as much context as you have: what the user reported, the course name, and any Moodle URLs.

Tip: Pasting a moodle.tru.ca URL will automatically be detected and parsed for course/activity context.
2

Add course context (optional)

Use the toolbar buttons to upload additional context:

  • .mbz Backup — Upload a Moodle course backup for full structure analysis and health checks
  • Screenshot — Upload a screenshot of the problem. The Companion can analyze images visually.
  • Saved Page — Save a Moodle page (Cmd+S / Ctrl+S in your browser) and upload the HTML file for settings extraction
3

Follow the diagnostic flow

The Companion will guide you through investigation. Watch for the mode badge:

  • EXPLORE — Gathering information. It will suggest things for you to check and questions to ask the user.
  • DIAGNOSE — Analyzing the evidence and ranking possible causes.
  • RESOLVE — Providing fix instructions and next steps.
4

After resolution

Once the issue is resolved, a prompt will appear with next steps:

  • Save as Case — Add to the team's case database with tags and difficulty rating
  • Draft Reply — Generate a professional reply for TeamDynamix, tailored to the audience
  • Export — Download the session as a Markdown file for training or documentation
  • Share — Copy a link so a colleague can view the conversation
  • Draft KB Article — Have the Companion draft a knowledge base article about the issue

🔎 Search Tab — Knowledge Base

Search across all ingested documentation using natural language. Results are ranked by semantic relevance, not just keyword matching. You can filter by source and click "Diagnose this issue" on any result to start a diagnostic conversation with that context pre-loaded.

📋 Cases Tab — Case Tracking & Analytics

Browse and search past cases. Click any tag to filter. Switch to the Analytics view for aggregate insights across all cases: weekly trends, difficulty distribution, top modules, and common tags.

Use the Export CSV button to download all case data for reporting in Excel or importing into TeamDynamix.

Setup

Installation Guide

Step-by-step instructions to get the Companion running on your computer. Choose your operating system below. No prior experience with Terminal or command line is needed — just copy and paste each command exactly as shown.

Before you start: You will need an Anthropic API key to power the AI features. Get one at console.anthropic.com — you'll need to create an account and add billing. The cost is roughly $0.01–0.05 per diagnostic conversation.

Mac Installation

1

Open Terminal

Press Cmd + Space to open Spotlight, type Terminal, and press Enter. A window with a command prompt will appear. All the commands below are typed (or pasted) here.

2

Install Homebrew (Mac package manager)

If you're not sure whether you have Homebrew, run this first to check:

            
            brew --version

If you see a version number, skip to Step 3. If you see "command not found", install Homebrew by pasting this command and pressing Enter:

            
            /bin/bash -c "$(curl -fsSL https://raw.githubusercontent.com/Homebrew/install/HEAD/install.sh)"

Follow the on-screen prompts. It may ask for your Mac password (you won't see characters as you type — that's normal).

3

Install Python

Check if you already have Python 3.11 or later:

            
            python3 --version

If the version is 3.11 or higher, skip to Step 4. Otherwise, install it:

            
            brew install python
4

Install Node.js

Check if you have Node.js 18 or later:

            
            node --version

If the version is 18 or higher, skip to Step 5. Otherwise:

            
            brew install node
5

Download the project

Option A — From GitHub (if you have git):

            
            cd ~/Desktop

            git clone https://github.com/blamb/moodle-support-companion.git

            cd moodle-support-companion

Option B — From a Zip file:

  1. Download the Zip file and save it to your Desktop
  2. Double-click to extract it
  3. In Terminal, run:
            
            cd ~/Desktop/moodle-support-companion
6

Install backend dependencies

            
            cd backend

            pip3 install -r requirements.txt
Note: The first run downloads AI model files (~90MB) for the knowledge base search. This is a one-time download.
7

Install frontend dependencies

            
            cd ../frontend

            npm install
8

Set your API key

Go back to the backend folder and set your Anthropic API key. Replace your-key-here with your actual key:

            
            cd ../backend

            export ANTHROPIC_API_KEY="your-key-here"
Important: You need to run the export command every time you open a new Terminal window. To make it permanent, you can add it to your shell profile — ask your team lead if unsure.
9

Start the backend server

Make sure you are in the backend folder, then run:

            
            python3 -m uvicorn app.main:app --port 8000 --reload

You should see output ending with something like Uvicorn running on http://127.0.0.1:8000. Leave this Terminal window open.

10

Start the frontend (in a new Terminal window)

Open a new Terminal window (Cmd + N) and run:

            
            cd ~/Desktop/moodle-support-companion/frontend

            npm run dev

You should see output with a URL like http://localhost:5173. Leave this window open too.

11

Open the app

Open your web browser and go to:

            
            http://localhost:5173
You're ready! You should see the Moodle Support Companion with the LT&I logo and three tabs: Diagnose, Search, and Cases.

Stopping the app

To stop the app, press Ctrl + C in each of the two Terminal windows. To start it again later, repeat Steps 8–11 (set the API key, start backend, start frontend, open browser).

Windows Installation

1

Open PowerShell

Press the Windows key, type PowerShell, and click Windows PowerShell. A blue command window will appear. All the commands below are typed (or pasted) here.

Tip: To paste in PowerShell, right-click in the window.
2

Install Python

Check if you already have Python 3.11 or later:

            
            python --version

If the version is 3.11 or higher, skip to Step 3. Otherwise:

  1. Go to python.org/downloads
  2. Click the big yellow "Download Python" button
  3. Run the installer
  4. IMPORTANT: Check the box that says "Add Python to PATH" at the bottom of the installer before clicking Install
  5. After installation, close and reopen PowerShell
Don't skip the PATH checkbox! If you miss it, the python command won't work. You'd need to uninstall and reinstall Python with the box checked.
3

Install Node.js

Check if you have Node.js 18 or later:

            
            node --version

If the version is 18 or higher, skip to Step 4. Otherwise:

  1. Go to nodejs.org
  2. Click the LTS (Long Term Support) download button
  3. Run the installer with all default options
  4. Close and reopen PowerShell after installation
4

Install Git (only if downloading from GitHub)

Check if you have git:

            
            git --version

If you see a version number, great. If not, go to git-scm.com/download/win, download and install with all defaults. Then close and reopen PowerShell.

If you're using the Zip file instead, you can skip this.

5

Download the project

Option A — From GitHub:

            
            cd ~\Desktop

            git clone https://github.com/blamb/moodle-support-companion.git

            cd moodle-support-companion

Option B — From a Zip file:

  1. Download the Zip file and save it to your Desktop
  2. Right-click the Zip → Extract All → Extract
  3. In PowerShell, run:
            
            cd ~\Desktop\moodle-support-companion
6

Install backend dependencies

            
            cd backend

            pip install -r requirements.txt
Note: If pip isn't recognized, try pip3 instead. The first run downloads AI model files (~90MB) — this is a one-time download.
7

Install frontend dependencies

            
            cd ..\frontend

            npm install
8

Set your API key

Go back to the backend folder and set your Anthropic API key. Replace your-key-here with your actual key:

            
            cd ..\backend

            $env:ANTHROPIC_API_KEY="your-key-here"
Important: You need to run the $env: command every time you open a new PowerShell window. To make it permanent, search for "Environment Variables" in Windows Settings and add ANTHROPIC_API_KEY as a user variable.
9

Start the backend server

Make sure you are in the backend folder, then run:

            
            python -m uvicorn app.main:app --port 8000 --reload

You should see output ending with Uvicorn running on http://127.0.0.1:8000. Leave this PowerShell window open.

Note: If python isn't recognized, try python3 instead.
10

Start the frontend (in a new PowerShell window)

Open a new PowerShell window and run:

            
            cd ~\Desktop\moodle-support-companion\frontend

            npm run dev

You should see a URL like http://localhost:5173. Leave this window open too.

11

Open the app

Open your web browser and go to:

            
            http://localhost:5173
You're ready! You should see the Moodle Support Companion with the LT&I logo and three tabs: Diagnose, Search, and Cases.

Stopping the app

To stop the app, press Ctrl + C in each of the two PowerShell windows. To start it again later, repeat Steps 8–11 (set the API key, start backend, start frontend, open browser).

Troubleshooting

"No module named app" when starting the backend
Make sure you are in the backend folder when you run the uvicorn command. Run pwd (Mac) or Get-Location (Windows) to check — it should end with /backend or \backend.
"Could not resolve authentication method" or API key errors
Your API key isn't set. Run the export (Mac) or $env: (Windows) command from Step 8 again in the same terminal window where you start the backend. Make sure there are no extra spaces in the key.
"Cannot connect to the backend" in the app
The backend server isn't running. Check the first Terminal/PowerShell window. If it closed or shows an error, go back to Step 9 and start it again.
The app loads but shows a blank page
Make sure you ran npm install in the frontend folder (Step 7) and that the frontend dev server is running (Step 10). Check the second Terminal/PowerShell window for errors.
pip or npm command not found
Close and reopen your Terminal/PowerShell after installing Python or Node.js. On Windows, make sure you checked "Add Python to PATH" during installation. On Mac, try pip3 instead of pip.
Port already in use
Another program is using port 8000 or 5173. Either close that program, or use a different port. For the backend: python3 -m uvicorn app.main:app --port 8001 --reload. You'll also need to update the frontend proxy settings if you change the backend port.

Technical Details

Architecture

For those interested in the technical underpinnings. 

┌─────────────────────────────────┐
│  React + Tailwind + TypeScript  │  ← Frontend (Vite, port 5173)
└───────────────┬─────────────────┘
                │ SSE streaming
┌───────────────┴─────────────────┐
│          FastAPI Backend         │  ← API server (port 8000)
├──────────┬──────────┬───────────┤
│ ChromaDB │ SQLite   │ Claude    │
│ vectors  │ cases    │ API       │
│ (384d)   │ + FTS5   │ (Sonnet)  │
└──────────┴──────────┴───────────┘

Backend

Python / FastAPI with Server-Sent Events for real-time streaming. Uvicorn ASGI server with hot-reload for development.

Vector Store

ChromaDB with all-MiniLM-L6-v2 sentence-transformer embeddings (384 dimensions). 1000-character chunks with 200-character overlap.

Case Database

SQLite with FTS5 full-text search. Automatic triggers keep the search index in sync. Cases include conversation history as JSON.

AI

Anthropic Claude API (Sonnet) with vision support for screenshot analysis. Custom system prompt enforcing the diagnostic methodology.

Frontend

React 19 with TypeScript, Tailwind CSS v4, and Vite. Custom hooks for conversation lifecycle and debounced search. SSE streaming for real-time responses.

Knowledge Pipeline

Ingests from 4 source types: MediaWiki HTML, WordPress WXR XML, DOCX, and Moodle page exports. Recursive text chunking with file-hash deduplication.
